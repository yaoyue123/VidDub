<task type="auto">
  <name>Task 2: Create discoveryStore Pinia store and register in stores/index.ts</name>
  <files>

  </files>
  <action>
    Create frontend/src/stores/discoveryStore.ts following the exact pattern of taskStore.ts (Composition API Pinia store with `defineStore`).

    Store name: 'discovery'

    State (all refs):
    - `sources` — ref<DiscoverySourceItem[]>([])
    - `searchResults` — ref<DiscoveryItem[]>([])
    - `scanResults` — ref<DiscoveryResultItem[]>([])
    - `loading` — ref(false)
    - `searchLoading` — ref(false)
    - `activeTab` — ref<'search' | 'keywords' | 'channels'>('search')
    - `filters` — reactive({ minViews: null as number|null, maxViews: null as number|null, minDuration: null as number|null, maxDuration: null as number|null })
    - `searchQuery` — ref('')
    - `error` — ref<string | null>(null)

    Actions (all async):
    1. `search(query: string, maxResults = 20)` — sets searchLoading=true, calls discoveryApi.search(), stores result in searchResults, catches errors into `error` ref, finally sets searchLoading=false.
    2. `fetchSources()` — sets loading=true, calls discoveryApi.listSources(), stores result in sources, catches errors into `error` ref.
    3. `addSource(body: DiscoverySourceCreateBody)` — calls discoveryApi.createSource(body), then calls fetchSources() to refresh.
    4. `updateSource(id: number, body: DiscoverySourceUpdateBody)` — calls discoveryApi.updateSource(id, body), then calls fetchSources().
    5. `removeSource(id: number)` — calls discoveryApi.deleteSource(id), then calls fetchSources().
    6. `triggerScan(id: number)` — calls discoveryApi.triggerScan(id), then calls fetchSources() to update last_scanned_at.
    7. `fetchResults(params?: { source_id?: number; status?: string })` — sets loading=true, calls discoveryApi.listResults(params), stores result in scanResults.
    8. `ignoreResult(id: number)` — calls discoveryApi.ignoreResult(id), then calls fetchResults().
    9. `addToPipeline(youtubeUrl: string)` — imports dubApi from '@/api' locally, calls dubApi.create({ youtube_url: youtubeUrl }). Returns the response data for the caller to navigate. Catches errors into `error` ref and re-throws.
    10. `clearSearch()` — resets searchResults to [], searchQuery to '', error to null.
    11. `setActiveTab(tab: 'search' | 'keywords' | 'channels')` — sets activeTab.

    Error handling pattern: match taskStore.ts — catch errors, set `error.value` to a user-friendly string (e.g., error.response?.data?.detail || error.message), do NOT re-throw for data-fetching actions (fetchSources, fetchResults, etc.), DO re-throw for user-triggered actions (addSource, removeSource, addToPipeline) so components can show toasts.

    Import pattern:
    ```typescript
    import { defineStore } from 'pinia'
    import { ref, reactive } from 'vue'
    import { discoveryApi, dubApi } from '@/api'
    import type { DiscoveryItem, DiscoverySourceItem, DiscoverySourceCreateBody, DiscoverySourceUpdateBody, DiscoveryResultItem } from '@/api'
    ```

    In stores/index.ts, add:
    ```typescript
    export { useDiscoveryStore } from './discoveryStore'
    ```
    Place it alphabetically in the export list (before useTaskStore, after useConfigStore).
  </action>
