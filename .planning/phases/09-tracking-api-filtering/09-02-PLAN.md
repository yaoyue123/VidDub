  <action>
    Add a FastAPI streaming proxy endpoint that fetches thumbnails from `i.ytimg.com` and serves them with CORS headers. Per D-v5.1-06 (thumbnail proxy via FastAPI streaming) and BUG-01.

    1. **Add imports** at the top of `discovery.py`:
       ```python
       from fastapi.responses import StreamingResponse
       import httpx
       ```
       Note: `httpx` is already in requirements.txt. No new dependency.

    2. **Add the endpoint**:
       ```python
       @router.get("/thumbnail/{video_id}")
       async def proxy_thumbnail(video_id: str):
           """Proxy YouTube video thumbnail with CORS headers.

           Fetches from i.ytimg.com and streams to client, adding
           Cache-Control and CORS headers that YouTube's CDN omits.
           """
           url = f"https://i.ytimg.com/vi/{video_id}/hqdefault.jpg"

           async with httpx.AsyncClient(timeout=10.0) as client:
               try:
                   resp = await client.get(url)
                   resp.raise_for_status()
               except httpx.HTTPStatusError as e:
                   # YouTube may return 404 for non-existent video_ids
                   raise HTTPException(
                       status_code=resp.status_code if hasattr(e, 'response') else 502,
                       detail=f"Thumbnail fetch failed for {video_id}",
                   )
               except httpx.RequestError as e:
                   raise HTTPException(
                       status_code=502,
                       detail=f"Failed to connect to thumbnail server: {e}",
                   )

           return StreamingResponse(
               resp.aiter_bytes(),
               media_type=resp.headers.get("content-type", "image/jpeg"),
               headers={
                   "Cache-Control": "public, max-age=86400",
                   "Access-Control-Allow-Origin": "*",
               },
           )
       ```

    3. **Video ID validation**: The path parameter `video_id` is a string matched by FastAPI's default path converter. YouTube video IDs are 11 characters of [0-9a-zA-Z_-]. No strict validation needed at this level -- invalid IDs will return 404 from i.ytimg.com, which is proxied back to the client.

    Key decisions:
    - Use `httpx.AsyncClient` for non-blocking HTTP fetch -- matches the async FastAPI pattern.
    - Use `StreamingResponse` with `resp.aiter_bytes()` to stream without buffering entire file in memory. Large thumbnails won't consume server memory.
    - `Cache-Control: public, max-age=86400` caches for 24 hours -- thumbnails rarely change for the same video_id.
    - `Access-Control-Allow-Origin: *` fixes the CORS issue that prevents the frontend from loading i.ytimg.com images.
    - Timeout set to 10 seconds -- prevents hanging on slow thumbnail CDN responses.
    - No server-side caching of thumbnails (CDN caching is sufficient). Keeps implementation simple and avoids disk usage.
    - Media type falls back to `image/jpeg` if YouTube doesn't send content-type (unlikely but defensive).
  </action>
