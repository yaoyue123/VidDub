### 学术研究来源

- **SAGE Open (2025)**: 记录了 B 站上 YouTube 粉丝字幕组生态的崛起

| 类型 | 适配度 | 原因 | 代表案例 |

### 关键改编要求
1. **中文字幕是刚需** — 嵌入式双语字幕是标准预期，机器翻译字幕是"即时信誉杀手"
2. **中文配音优选项** — 长内容(纪录片/系列)强烈偏好专业中文配音
3. **剪掉西方赞助** — 删除中国不可用服务的推广
4. **B 站原生 CTA** — 用"投币/收藏/点赞"代替"YouTube Subscribe"
5. **缩略图本地化** — 英文文字替换为中文

## 2. 视频传播力预测算法

### 学术前沿
- **多模态病毒传播预测 (IGI Global, 2025)**: 127,843 视频数据集，89.7% 准确率，超越单模态模型 9-13%
- **时态-病毒联合推荐 (MDPI Electronics, 2025)**: 中国传媒大学+北大+清华联合研究，基于抖音/B 站数据

### 四个关键预测因子
1. **观看速度 (权重最高)** — 前 24 小时内每小时观看增量是最大单一预测变量
2. **互动率** — 点赞/评论/观看比，>10% 为高互动
3. **内容情感** — 正向情感+高亮度画面提升分享率 18%
4. **时效性** — 发布 24 小时内是病毒传播的黄金窗口

### 实践工具
- **Apify YouTube Channel Intelligence**: 0-100 分评分算法，四维度加权
  - 观看速度 (40pts)
  - 互动率 (30pts)
  - 评分质量 (20pts)
  - 时效性加分 (10pts)

## 3. 翻译适配度评估标准

### 低翻译成本内容（高分）
- 视觉驱动内容（健身、动画）— 对白少
- 英语教学内容 — 本身就是语言学习材料
- 结构化演讲（TED 风格）— 清晰的句子边界
- 单一说话人 — 不需要多人音色

### 高翻译成本内容（低分）
- 多人对话 — 需要说话人分离 + 多音色
- 密集双关/梗 — 文化依赖度高
- 即兴对话 — 碎句、打断、重叠
- 有大量屏幕文字的教程 — 需要同步翻译屏幕文字

## 4. 内容规则引擎设计参考

### 常见搬运策略模式

| 策略名称 | 条件组合 | 适用场景 |
|----------|---------|---------|
| "爆款优先" | 播放量 > 50万 + 发布时间 < 7天 + 类型不限 | 追求流量最大化 |
| "教育精品" | 类型=教育 + 时长 5-25min + 互动率 > 5% | 垂直细分深耕 |
| "科技快报" | 类型=科技 + 时长 < 15min + 发布时间 < 3天 | 时效性强的科技新闻 |
| "低风险搬运" | 类型≠政治/宗教 + 播放量 > 1万 + 语言=英语 | 避免内容审核风险 |
| "测试水温" | 综合评分 > 70 + 制作成本 > 8 | 只搬运高质量且容易做的 |

## 5. 对本项目的建议

### 评分引擎架构
```
视频输入 → YouTube Data API / RSS
    ├─ 传播潜力分析 (view velocity + engagement + recency)
    ├─ 翻译适配评估 (language density + clarity + cultural fit)
    ├─ 内容质量评估 (duration + authority + category match)
    ├─ 中文市场匹配 (category-Bilibili hot zone mapping)
    └─ 制作成本预估 (captions available + speaker count + edit needs)
    → 加权综合分 (用户可调权重) → 排序推荐
```

### 推荐技术方案
- **YouTube Data API v3** 作为主数据源（获取观看量/互动/发布信息）
- **yt-dlp** 作为降级方案（API Key 不可用时用 RSS 抓取）
- **SiliconFlow Chat API** 用于内容分类和文化适配评估（复用现有集成）
- **本地计算** 用于时效性、时长、去重等基础评分（减少 API 调用）
- **SQLite** 存储评分历史和规则配置（一致性）
