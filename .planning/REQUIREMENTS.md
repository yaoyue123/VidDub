# You2Bili - 需求文档 (v4.0)

## v4.0 智能选题与内容追踪

### 核心理念

从"被动加工"升级为"主动发现"。管道已经很完善（下载→配音→发布），但现在的问题是：**搬什么视频？**

答案不是拍脑袋，而是建立一套**可量化、可自定义、自动运行**的内容评分系统。

### 背景：什么内容适合搬

基于学术研究和行业实践的总结：

| 内容类型 | 适配度 | 核心适配工作 |
|----------|--------|-------------|
| 知识科普/教育 | ⭐⭐⭐⭐⭐ | 中文字幕 + 专业术语本地化 |
| 科技/数码评测 | ⭐⭐⭐⭐ | 剪辑西方赞助片段 + 中文字幕 |
| 动画/视觉内容 | ⭐⭐⭐⭐⭐ | 几乎不需要改编 |
| 健身/运动 | ⭐⭐⭐⭐⭐ | 屏幕文字翻译，对白需求低 |
| 音乐表演 | ⭐⭐⭐⭐ | 语言障碍最小 |
| 游戏 | ⭐⭐⭐⭐ | 对接中文游戏社区梗 |
| 英语教学 | ⭐⭐⭐⭐⭐ | 中国巨大英语学习需求 |
| 娱乐/喜剧 | ⭐⭐⭐ | 需要高质量文化改编 |
| 商业/动机 | ⭐⭐⭐ | 需要完全文化重构 |

### 评分维度（五维模型）

每个视频从五个维度打分，产生一个综合分（0-100）：

| 维度 | 权重 | 衡量指标 |
|------|------|---------|
| **传播潜力** | 30% | 观看速度（每小时播放增量）、互动率（点赞/评论/观看比）、发布时间（24h内加分） |
| **翻译适配度** | 25% | 语言密度（字幕占比）、对白清晰度、文化依赖度 |
| **内容质量** | 20% | 时长区间（5-20min最佳）、频道权威性、内容类型匹配度 |
| **中文市场潜力** | 15% | 内容类型与 B 站热门匹配度、是否涉及中国相关话题 |
| **制作成本** | 10% | 是否有内置字幕、是否需要大量剪辑、是否需要语音分离 |

### 自定义规则引擎

用户不是被动的——他们可以定义自己的筛选逻辑：

```
规则示例 1: "我只关心科技类视频，播放量 > 10万，时长 < 30分钟"
规则示例 2: "每天给我推荐 5 个评分最高的英语教学视频"
规则示例 3: "排除政治/宗教内容，只看 7 天内发布的"
```

## Requirements

### Phase 13: 选题评分引擎

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| SC-01 | 五维评分模型 | P0 | 实现传播潜力/翻译适配度/内容质量/中文市场潜力/制作成本五维评分 |
| SC-02 | 观看速度分析 | P0 | 通过 YouTube API 获取每小时播放增量，计算传播潜力分 |
| SC-03 | 互动率分析 | P0 | 计算点赞/评论/观看比，评估内容吸引力 |
| SC-04 | 翻译适配度评估 | P0 | 分析字幕占比、语言密度、对白清晰度，判断翻译难度和效果 |
| SC-05 | 内容类型识别 | P0 | 通过 LLM 分类视频内容类型（科技/教育/娱乐/游戏等） |
| SC-06 | 时长适配评估 | P1 | 5-20分钟为最佳搬运区间，过长过短降分 |
| SC-07 | 中文市场匹配 | P1 | 内容类型与 B 站热门分区匹配度评分 |
| SC-08 | 批量评分 | P1 | 对频道/搜索结果批量打分排序 |
| SC-09 | 评分历史存储 | P1 | 保存评分记录，支持对比和趋势 |

### Phase 14: 智能发现引擎

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| DS-01 | 智能频道推荐 | P0 | 基于用户已搬运内容，推荐相似优质频道 |
| DS-02 | 关键词主题挖掘 | P0 | 根据热门话题自动生成搜索关键词 |
| DS-03 | 趋势视频抓取 | P0 | 抓取 YouTube Trending/热门分类视频 |
| DS-04 | 频道定时扫描 | P1 | 定时扫描关注的频道，自动评分新视频 |
| DS-05 | 发现源管理 | P1 | 添加/编辑/删除发现源（频道/关键词/分类） |
| DS-06 | 去重机制 | P1 | 已搬运/已评分的视频自动去重，不重复推荐 |
| DS-07 | 每日推荐列表 | P1 | 每天生成一份"今日推荐"视频列表 |

### Phase 15: 自定义规则引擎

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| RE-01 | 可视化规则构建器 | P0 | 在 Web 界面用条件组合构建筛选规则 |
| RE-02 | 多维度条件 | P0 | 支持按评分维度、播放量、时长、发布时间、内容类型设置条件 |
| RE-03 | 规则权重调整 | P0 | 用户可以调整五维评分的权重比例 |
| RE-04 | 白名单/黑名单 | P1 | 支持频道白名单（只关注这些）和关键词黑名单（排除这些） |
| RE-05 | 规则模板 | P1 | 内置 5 套预设规则模板（科技搬运/英语教学/娱乐搞笑/综合/自定义） |
| RE-06 | 规则测试 | P1 | 用最近 50 个视频测试规则效果，看会筛出什么 |
| RE-07 | 规则保存与切换 | P1 | 保存多套规则，一键切换 |
| RE-08 | 自动执行 | P2 | 规则匹配后自动创建下载任务（可选开启） |

### Phase 16: 内容追踪仪表盘

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| TK-01 | 推荐视频列表页 | P0 | 展示评分排序的视频列表，支持筛选和预览 |
| TK-02 | 频道追踪面板 | P0 | 展示已追踪频道的关键指标（新视频数/平均评分/最新视频） |
| TK-03 | 趋势热榜 | P1 | 展示当前热门话题和趋势视频 |
| TK-04 | 评分详情弹窗 | P1 | 点击视频展开五维评分详情，展示每个维度的得分和原因 |
| TK-05 | 一键搬运 | P1 | 从推荐列表直接创建配音任务 |
| TK-06 | 排行榜 | P2 | 展示历史搬运中最成功的视频 TOP 榜 |

### Phase 17: 数据分析与优化

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| AN-01 | 搬运效果追踪 | P1 | 跟踪已搬运视频在国内平台的播放/互动数据 |
| AN-02 | 评分模型反馈 | P1 | 对比预测评分 vs 实际表现，持续优化评分模型 |
| AN-03 | 月度搬运报告 | P2 | 每月生成搬运效果报告 |

## 非功能需求 (v4.0)

| ID | 需求 | 说明 |
|----|------|------|
| NF-SC-01 | 评分速度 | 单个视频评分 < 3 秒 |
| NF-SC-02 | 批量处理 | 单频道扫描（50 视频）< 2 分钟 |
| NF-SC-03 | YouTube API 配额管理 | 智能缓存，减少 API 调用次数 |
| NF-SC-04 | 离线评分 | 部分评分维度（内容类型、翻译适配度）支持离线计算 |
| NF-SC-05 | API Key 可选 | YouTube Data API Key 是可选的——不用 API 时用 RSS/yt-dlp 降级方案 |

## 数据模型扩展

### 新增表

**video_scores（视频评分表）**
```
id, video_id, youtube_id, title, channel, channel_id,
传播潜力分, 翻译适配分, 内容质量分, 市场潜力分, 制作成本分, 综合分,
评分时间, 评分版本, 各维度详情(JSON)
```

**content_rules（自定义规则表）**
```
id, name, enabled, conditions(JSON), weights(JSON),
whitelist_channels(JSON), blacklist_keywords(JSON),
created_at, updated_at
```

**discovery_sources（发现源表）**
```
id, type(channel/keyword/category), source_value,
enabled, last_scan_at, scan_interval_hours,
created_at
```

**tracked_channels（追踪频道表）**
```
id, youtube_channel_id, channel_name, thumbnail_url,
subscriber_count, video_count, avg_score,
last_scanned_at, is_active, created_at
```

## 技术依赖

- YouTube Data API v3（可选 — 有 API Key 时优先用，没有时降级到 RSS/yt-dlp）
- yt-dlp（已有）
- SiliconFlow Chat API（内容分类 + 中文市场匹配评估）
- SQLite（评分记录 + 规则存储）
